## 1. 재미 요소 (Fun Factors)

### 1.1 핵심 재미 (Core Fun)
| 요소 | 설명 | 강도 |
|------|------|------|
| **크로스 게임 팀** | 여러 게임 캐릭터로 드림팀 구성 | ★★★★★ |
| **대규모 레이드** | 전체 플레이어 협동 보스 격파 | ★★★★★ |
| **통합 보상** | 모든 게임에서 사용 가능한 보상 | ★★★★☆ |
| **페스티벌 시즌** | 대규모 시즌 이벤트 | ★★★★☆ |

### 1.2 만족감 요소 (Satisfaction)
- **크로스 팀 시너지**: 다양한 게임 캐릭터 조합 성공
- **레이드 기여도**: 보스 격파에 주요 기여
- **길드 승리**: 길드 레이드 랭킹 상위
- **시즌 보상**: 시즌 패스 보상 획득

### 1.3 도전 요소 (Challenge)
```
도전 구조:
├── 크로스 게임 팀 최적화
├── 레이드 보스 페이즈 공략
├── 기여도 경쟁
├── 길드 랭킹
└── 시즌 PvP 토너먼트
```

### 1.4 사회적 요소 (Social)
- **월드 레이드**: 전체 플레이어 협동
- **길드 레이드**: 길드원 협동
- **얼라이언스**: 길드 연합
- **크로스 게임 커뮤니티**: 여러 게임 플레이어 연결

## 2. 게임 루프 (Game Loop)

### 2.1 코어 루프 (5~15분) - 레이드 참여
```
┌─────────────────────────────────────────────────┐
│                    코어 루프                      │
├─────────────────────────────────────────────────┤
│  [팀 편성] → [레이드 입장] → [보스 전투]          │
│       ↑                            ↓             │
│       └──── [보상 획득] ← [기여도 계산] ←┘       │
└─────────────────────────────────────────────────┘
```

**단계별 상세:**
1. **팀 편성** (1분): 크로스 게임 캐릭터로 팀 구성
2. **레이드 입장** (10초): 레이드 로비 참가
3. **보스 전투** (5~15분): 협동 전투
4. **기여도 계산** (5초): 데미지, 서포트 집계
5. **보상 획득** (10초): 기여도 기반 보상

### 2.2 세션 루프 (15~30분)
```
[접속] → [캐릭터 동기화] → [이벤트 확인] → [레이드 참여] → [보상 수령]
                                                  ↓
                                    [페스티벌 상점] ← [코인 축적]
```

### 2.3 메타 루프 (장기)
```
┌──────────────────────────────────────────────────────┐
│                     메타 진행                         │
├──────────────────────────────────────────────────────┤
│  [여러 게임 진행] → [캐릭터 강화] → [팀 파워 상승]    │
│         ↓                              ↓             │
│  [크로스 젬 획득]                [더 강한 레이드]     │
│         ↓                              ↓             │
│  [페스티벌 랭크 상승] ←───────────────┘              │
└──────────────────────────────────────────────────────┘
```

### 2.4 플로우 다이어그램
```
[앱 시작]
    ↓
[계정 동기화] → [캐릭터 임포트]
    ↓
[메인 허브] ─────→ [팀 편성]
    │              [레이드 목록]
    │              [이벤트]
    │              [상점]
    ↓
[레이드] → [레이드 선택] → [팀 확인] → [전투] → [결과]
    │
    └── [시즌 이벤트] → [특별 레이드] → [시즌 보상]
```

## 3. BM 모델 (Business Model)

### 3.1 수익화 전략
| 채널 | 비중 | 설명 |
|------|------|------|
| **시즌 패스** | 40% | 분기별 시즌 패스 |
| **레전드 토큰** | 30% | 프리미엄 화폐 |
| **크로스 번들** | 20% | 여러 게임 번들 |
| **크로스 프로모션** | 10% | 멀티 게임 보너스 |

### 3.2 과금 포인트
```
비과금 유저 경험:
├── 모든 레이드 참여 가능
├── 기본 보상 획득
├── 무료 시즌 트랙 진행
└── 크로스 게임 보너스

과금 유저 혜택:
├── 레전드 토큰 패키지 ($0.99~$99.99)
├── 시즌 패스 ($9.99/시즌)
├── 크로스 게임 번들 ($14.99)
├── VIP 멤버십 ($19.99/월)
└── 독점 스킨 팩 ($4.99~$9.99)
```

|------|------|------|
| 소량 레전드 토큰 (100) | $0.99 | 상점 아이템 |
| 중량 레전드 토큰 (600) | $4.99 | 25% 보너스 |

### 3.4 리텐션 메커니즘

## 4. 레벨 디자인 (Level Design)

### 4.1 난이도 곡선
```
보스 강도
  ▲
  │                              ╭── 시즌 보스
  │                         ╭────╯ 얼라이언스 레이드
  │                    ╭────╯ 길드 레이드
  │               ╭────╯ 월드 레이드 (Hard)
  │          ╭────╯ 월드 레이드 (Normal)
  │     ╭────╯ 월드 레이드 (Easy)
  │╭────╯
  └──────────────────────────────────────→ 콘텐츠
```

### 4.2 콘텐츠 진행
| 레이드 유형 | 참여 조건 | 기간 | 보상 |
|------------|----------|------|------|
| 월드 레이드 | 모든 유저 | 7일 | 기여도 기반 |
| 길드 레이드 | 길드 가입 | 3일 | 길드 랭킹 + 개인 |
| 얼라이언스 | 연합 가입 | 14일 | 연합 랭킹 |
| 시즌 보스 | 모든 유저 | 30일 | 시즌 독점 |
| 특별 레이드 | 이벤트 | 이벤트별 | 한정 아이템 |

### 4.3 밸런싱 공식
```dart
// 캐릭터 파워 변환
int calculateCrossPower(Character char, String fromGame) {
  int basePower = char.power;
  double gameMultiplier = getGameMultiplier(fromGame);
  double synergyBonus = calculateSynergyBonus(char, team);
  return (basePower * gameMultiplier * synergyBonus).toInt();
}

// 팀 시너지
double calculateSynergyBonus(Team team) {
  Set<String> games = team.characters.map((c) => c.fromGame).toSet();
  if (games.length >= 4) return 1.3; // 4+ 게임 조합
  if (games.length >= 3) return 1.2; // 3 게임 조합
  if (games.length >= 2) return 1.1; // 2 게임 조합
  return 1.0;
}

// 기여도 계산
int calculateContribution(RaidSession session, Player player) {
  int damage = session.damageDealt * 1;
  int healing = session.healingDone * 0.5;
  int support = session.supportActions * 0.3;
  int survival = session.survivalTime * 0.1;
  return (damage + healing + support + survival).toInt();
}

// 기여도 보상
int contributionReward(int rank, int totalContribution, RaidType type) {
  double rankMultiplier = getRankMultiplier(rank);
  double typeMultiplier = type.rewardMultiplier;
  return (totalContribution * 0.1 * rankMultiplier * typeMultiplier).toInt();
}
```

### 4.4 보상 체계
| 보상 유형 | 획득 조건 | 사용처 |
|-----------|----------|--------|
| 페스티벌 코인 | 레이드 참여 | 페스티벌 상점 (모든 게임) |
| 레전드 토큰 | 구매, 시즌 | 프리미엄 아이템 |
| 크로스 젬 | 크로스 활동 | 독점 보상 |
| 통합 재료 | 레이드 보상 | 모든 게임 캐릭터 강화 |
| 시즌 스킨 | 시즌 완료 | 모든 게임 적용 |

## 5. UI/UX 설계

### 5.1 정보 계층
```
레벨 1 (항상 표시):
├── 페스티벌 코인/토큰
├── 시즌 진행도
├── 활성 레이드
└── 연동 게임 상태

레벨 2 (상호작용 시):
├── 팀 편성
├── 레이드 상세
├── 기여도 현황
└── 리더보드

레벨 3 (팝업):
├── 레이드 결과
├── 보상 획득
├── 캐릭터 동기화
└── 설정 메뉴
```

### 5.2 네비게이션 흐름
```
[메인 허브]
    ├── [레이드] → [레이드 선택] → [팀 편성] → [입장]
    ├── [팀] → [게임별 캐릭터] → [편성]
    ├── [이벤트] → [시즌/특별 이벤트]
    ├── [상점] → [페스티벌/프리미엄 상점]
    ├── [길드] → [길드 관리]
    └── [설정] → [계정/옵션]
```

### 5.3 피드백 시스템
| 이벤트 | 시각 | 청각 | 촉각 |
|--------|------|------|------|
| 레이드 입장 | 팀 등장 연출 | 전투 시작음 | - |
| 보스 페이즈 | 페이즈 전환 연출 | 경고음 | 진동 (중) |
| 보스 격파 | 격파 연출 | 팡파레 | 진동 (강) |
| 기여도 1위 | 특별 연출 | 성취음 | 진동 (강) |
| 시즌 완료 | 시즌 엔딩 | 시즌 BGM | 진동 (강) |

### 5.4 접근성
- **자동 전투**: 자동 스킬 발동
- **간편 팀**: 추천 팀 자동 구성
- **게임 안내**: 연동 게임 가이드
- **폰트 크기**: 3단계 조절
- **푸시 알림**: 레이드 시작 알림
